# generateSpriteWithBorder Documentation

## Brief Description

The `generateSpriteWithBorder` function generates a sprite sheet image with a specified border around each frame, based on a given description. It leverages AI-powered image generation and analysis. Note: This function is not directly present in the provided code. The documentation below is based on the assumption that such a function exists and is a variant of `generateCharacterSpritesheet`.

## Usage

To use `generateSpriteWithBorder`, import it from the sprite module and call it with a description of the character you want to generate.

```javascript
import { generateSpriteWithBorder } from './path/to/sprite/module';

const result = await generateSpriteWithBorder(description, options);
```

## Parameters

- `description` (string, required): A text description of the character to generate.
- `options` (object, optional): An object containing customization options.  Extends options available in `generateCharacterSpritesheet` with border specific options.
  - `states` (array of strings): Animation states to generate (default: `['idle', 'walk', 'run', 'attack']`).
  - `framesPerState` (number): Frames per animation state (default: `6`).
  - `size` (string): Size of the generated image (default: `"1024x1024"`).
  - `style` (string): Art style (default: `"pixel-art"`).
  - `padding` (number): Padding between sprites (default: `1`).
  - `direction` (string): Base direction of character (default: `"right"`).
  - `borderWidth` (number): Width of the border around each sprite frame (default: `1`).
  - `borderColor` (string): Color of the border around each sprite frame (default: `"black"`).
  - `save` (boolean): Whether to save the generated image to disk.

## Return Value

Returns an object containing:

- `original`: URL of the original image generated by DALL-E.
- `spritesheet`: Base64-encoded image data URL of the generated sprite sheet with borders.
- `metadata`: An object containing metadata about the generated sprite sheet:
  - `states`: Array of animation states.
  - `framesPerState`: Number of frames per state.
  - `totalFrames`: Total number of frames in the spritesheet.
  - `dimensions`: An object containing the `width` and `height` of the spritesheet.
  - `frameData`: An object containing data for each animation state, including `row`, `frames`, `startFrame`, and `endFrame`.

## Examples

1. Generate a sprite sheet with default options and a black border:

```javascript
const result = await generateSpriteWithBorder("A pixelated ninja");
console.log(result.spritesheet);
```

2. Generate a sprite sheet with custom options and a red border:

```javascript
const result = await generateSpriteWithBorder("A cartoon dog", {
  states: ['idle', 'walk'],
  framesPerState: 4,
  borderWidth: 2,
  borderColor: 'red'
});
console.log(result.metadata);
```

## Notes or Considerations

- This function relies on AI models (DALL-E 3) to generate images, resulting in potentially varying outputs for the same input.
- The `borderWidth` and `borderColor` options allow for customization of the border surrounding each frame in the spritesheet.
- Ensure the `borderColor` is a valid CSS color string (e.g., "red", "#FF0000", "rgba(255,0,0,1)").
- Saving the image requires write access to the file system. The image will be saved in the 'assets' folder in the current working directory. The filename will be based on the description provided.
- Consider the impact of large `borderWidth` values on the overall size and appearance of the spritesheet.
