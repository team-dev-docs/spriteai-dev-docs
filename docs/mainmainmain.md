
  
  # Dev-Docs VS Code Extension Quickstart Guide

Welcome to the Dev-Docs VS Code Extension! This guide will help you get started quickly and efficiently.

## Installation

1. Open Visual Studio Code.
2. Go to the Extensions view by clicking on the square icon in the left sidebar or pressing `Ctrl+Shift+X` (Windows/Linux) or `Cmd+Shift+X` (Mac).
3. Search for "Dev-Docs" in the Extensions view search bar.
4. Click on the "Install" button next to the Dev-Docs extension.

## Setting Up Your Project

1. Open your project folder in VS Code.
2. Create a new folder named `docs` in your project root if it doesn't already exist.

## Using the Extension

1. Open the command palette by pressing `Ctrl+Shift+P` (Windows/Linux) or `Cmd+Shift+P` (Mac).
2. Type "Dev-Docs" and select "Dev-Docs: Create Documentation" from the list of commands.
3. Choose the type of documentation you want to create (e.g., function, class, module).
4. Follow the prompts to enter the necessary information about your code.
5. The extension will generate a Markdown file in the `docs` folder with the documentation.

## Customizing Documentation Templates

1. Open the Dev-Docs extension settings in VS Code.
2. Look for the "Custom Templates" section.
3. Click on "Edit in settings.json" to customize the documentation templates according to your needs.

## Generating Documentation for Existing Code

1. Select the code you want to document in your file.
2. Right-click and choose "Generate Documentation" from the context menu.
3. The extension will analyze your code and create documentation based on the selected template.

## Viewing Generated Documentation

1. Navigate to the `docs` folder in your project.
2. Open the generated Markdown files to view your documentation.
3. Use VS Code's built-in Markdown preview to see how the documentation will look when rendered.

## Updating Documentation

1. As you make changes to your code, remember to update the corresponding documentation files.
2. You can use the "Generate Documentation" command again on updated code to refresh the documentation.

## Getting Help

If you encounter any issues or have questions:

1. Check the extension's documentation in the VS Code marketplace.
2. Visit the extension's GitHub repository for more detailed information and to report issues.
3. Reach out to the extension's support channels if available.

Happy documenting with Dev-Docs!
