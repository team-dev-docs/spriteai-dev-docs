

  # Understanding Parselmouth's Mapping Output

This guide explains the structure and contents of the JSON files generated by Parselmouth's package mapping process.

## Overview

Parselmouth generates JSON files that map Conda/PyPI package metadata to a structured format. The main output is an index file containing mappings for multiple packages, keyed by their SHA256 hashes.

## Index File Structure

The main index file has the following high-level structure:

```json
{
  "root": {
    "<package_sha256>": {
      // Package mapping data
    },
    "<another_package_sha256>": {
      // Another package's mapping data  
    }
    // More package entries...
  }
}
```

The `root` object contains mappings for multiple packages, with each package identified by its SHA256 hash.

## Package Mapping Structure

Each package mapping contains the following key information:

```json
{
  "name": "package_name",
  "version": "1.2.3",
  "build": "py39h123456_0",
  "subdir": "linux-64",
  "depends": [
    "python >=3.9,<3.10.0a0",
    "numpy >=1.16.6,<2.0a0"
  ],
  "constrains": [
    "scipy >=1.5.0"
  ],
  "files": [
    {
      "path": "lib/python3.9/site-packages/package_name/__init__.py",
      "size_in_bytes": 1234,
      "sha256": "abc123..."
    },
    // More file entries...
  ]
}
```

Key fields:

- `name`: The package name
- `version`: Package version
- `build`: Conda build string
- `subdir`: Conda subdirectory (platform)
- `depends`: List of package dependencies
- `constrains`: List of version constraints
- `files`: List of files included in the package, with paths, sizes, and hashes

## Interpreting the Mapping

This mapping provides several key pieces of information:

1. Package identification (name, version, build)
2. Platform compatibility (subdir)
3. Dependency requirements
4. Version constraints
5. Detailed file inventory

This information can be used for:

- Resolving package dependencies
- Verifying package contents
- Analyzing package structures
- Comparing packages across versions or platforms

## Accessing the Data

To work with this data programmatically, you can load the JSON file and access the mappings like this:

```python
import json

with open('index.json', 'r') as f:
    index_data = json.load(f)

# Access a specific package mapping
package_sha256 = 'abc123...'
package_mapping = index_data['root'][package_sha256]

# Example: Print package name and version
print(f"Package: {package_mapping['name']} v{package_mapping['version']}")

# Example: List dependencies
print("Dependencies:")
for dep in package_mapping['depends']:
    print(f"- {dep}")
```

This structure allows for efficient lookups and analysis of package metadata across large collections of packages.
